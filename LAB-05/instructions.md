# LAB-05: Agile Sprint Planning Simulation

**Duration:** 40 minutes  
**Points:** 4  
**Due:** Monday Week 2, End of Class

---

## 🎯 LEARNING OBJECTIVES

- Understand Agile/Scrum methodology
- Practice sprint planning with Trello
- Learn MoSCoW prioritization
- Write user stories
- Estimate story points

---

## 📋 REQUIREMENTS

Create a sprint plan for a Library Management System:

1. **User Stories (6 minimum):**
   - [ ] Write user stories using proper format
   - [ ] Format: "As a [role], I want [feature], so that [benefit]"
   - [ ] Include acceptance criteria for each

2. **MoSCoW Prioritization:**
   - [ ] Label stories: Must Have, Should Have, Could Have, Won't Have
   - [ ] At least 2 Must Have stories
   - [ ] Balanced priorities

3. **Story Point Estimation:**
   - [ ] Estimate each story (1, 2, 3, 5, 8, 13)
   - [ ] Total sprint capacity: 20-25 points
   - [ ] Justify your estimates

4. **Documentation:**
   - [ ] Create a markdown file documenting your sprint plan
   - [ ] Include all stories, priorities, and estimates

---

## 💻 EXAMPLE USER STORY

```
Title: Search Books by Title
Priority: Must Have
Story Points: 3

User Story:
As a library patron,
I want to search for books by title,
So that I can quickly find the book I'm looking for.

Acceptance Criteria:
- Search input accepts text
- Results display matching book titles
- Search is case-insensitive
- Shows "No results" if no matches
```
